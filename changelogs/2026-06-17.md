##### 17/06/2026

# Esportazione configurazioni di rilevamento e rebranding export AI

## Supporto completo ai parametri di detection nei flussi di export e reportistica.

### 🔄 Modifiche (Changed)

- Estesa `prepare_chart_data()` per includere nel payload:
  - `analysis_method`
  - `ruptures_config`
- Aggiunta serializzazione completa della configurazione Ruptures (`model`, `penalty`, `min_segment_sec`, `smooth_window_sec`, `min_cp_pct`, `merge_gap_sec`, `merge_power_diff_pct`, `min_effort_sec`, `opener_threshold_pct`).
- Aggiornata l'esportazione JSON standard per includere nei `detection_parameters` la configurazione realmente utilizzata per il rilevamento degli effort:
  - configurazione Legacy quando `method = legacy`
  - configurazione Ruptures quando `method = ruptures`
- Applicata la stessa logica anche all'export JSON destinato al training di modelli.
- Centralizzato il recupero delle configurazioni `EffortConfig` e `SprintConfig` tramite variabili locali (`effort_cfg`, `sprint_cfg`) evitando accessi ripetuti alla sessione.
- Rinominato il modello request:
  - `ExportJsonLlamaRequest` → `ExportJsonModelRequest`
- Rinominato endpoint API:
  - `/export/{session_id}/json-llama`
  - → `/export/{session_id}/json-Model`
- Aggiornata la documentazione inline e i commenti associati all'export AI da "Llama" a "Model".

### 🎨 Miglioramenti UI

- Aggiornata la sidebar di configurazione per distinguere visivamente il metodo di rilevamento utilizzato:
  - modalità **Legacy**
  - modalità **Ruptures**
- Aggiunta visualizzazione dettagliata dei parametri Ruptures direttamente nel pannello configurazione.
- Migliorata la sezione configurazione mostrando i valori effettivamente utilizzati dal metodo selezionato.
- Aggiornata l'interfaccia export:
  - "Export per Llama AI" → "Export per Model"
  - aggiornati ID HTML, variabili JavaScript, status indicator e messaggi di conferma.
- Aggiornato il modal di selezione attività per riflettere il nuovo naming "Model".

### ⚙️ Infrastruttura

- Uniformato il payload di esportazione affinché i parametri di rilevamento siano esportati in modo coerente indipendentemente dal metodo utilizzato.
- Aggiornato l'export pubblico dei moduli con modifica all'elenco `__all__`.

### 🐞 Correzioni (Fixed)

- Corretto l'export JSON affinché i parametri di detection riflettano il metodo realmente usato nell'analisi invece di esportare sempre la sola configurazione Legacy.
- Corretto il report PDF per mostrare un riepilogo coerente della configurazione di rilevamento:
  - dettaglio completo per Ruptures
  - riepilogo Legacy per il metodo tradizionale.

### Risultato finale

I file esportati, la sidebar applicativa e i report PDF riportano ora il metodo di rilevamento effettivamente utilizzato (Legacy o Ruptures) insieme ai relativi parametri completi, migliorando tracciabilità, riproducibilità delle analisi ed esportazione dei dati destinati al training dei modelli.
