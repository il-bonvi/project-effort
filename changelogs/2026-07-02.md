# Report HTML con integrazione della Mappa 2D

## Il report HTML esportato include ora una seconda scheda dedicata alla visualizzazione geografica della sessione, integrata nello stesso documento dell'altimetria.

### ✨ Nuove funzionalità (Added)

- Integrata la **Mappa 2D** direttamente nel report HTML esportato, accessibile tramite un sistema a schede senza dover aprire una pagina separata.
- Aggiunta la generazione automatica dei dati geografici della sessione (`_generate_map2d_data`), comprensiva di:
  - tracciato GeoJSON;
  - dati di effort e sprint;
  - dati del grafico altimetrico sincronizzato;
  - centro e livello di zoom iniziale della mappa.
- Introdotta la trasformazione automatica di `map2d.js` (`_make_map2d_export_js`) per consentirne l'esecuzione all'interno del report senza conflitti con gli script già presenti.
- Implementata la conversione del report HTML in una struttura a **due schede** (`_wrap_export_html_with_tabs`):
  - 📈 Altimetria
  - 🗺️ Mappa 2D
- Aggiunto un meccanismo di fallback che mantiene il report nella versione tradizionale quando non sono disponibili dati GPS o la mappa non può essere generata.

### 🔄 Modifiche (Changed)

- Esteso il processo di esportazione HTML per includere automaticamente la preparazione e l'integrazione della Mappa 2D quando la sessione contiene dati GPS validi.
- Il codice JavaScript della Mappa 2D viene ora isolato tramite IIFE, con rinomina automatica degli ID DOM condivisi e riesposizione controllata delle sole funzioni necessarie agli handler dinamici.
- Riorganizzato il layout del report esportato introducendo una barra di navigazione con cambio scheda e una struttura dedicata alla visualizzazione simultanea di Altimetria e Mappa 2D.
- Integrate automaticamente nel report le dipendenze Leaflet necessarie al funzionamento della mappa.

### 🐞 Correzioni (Fixed)

- Eliminati i conflitti tra gli elementi DOM della Mappa 2D e quelli già presenti nella vista Altimetria mediante prefissazione degli ID e isolamento dello scope JavaScript.
- Corretta l'inizializzazione della mappa quando il contenitore è inizialmente nascosto, evitando livelli di zoom errati al primo caricamento.
- Migliorata la gestione del ridimensionamento della mappa e del grafico altimetrico tramite `ResizeObserver`, garantendo un rendering corretto dopo l'attivazione della scheda o variazioni del layout.
- Aggiornata la gestione delle richieste ai tile provider Leaflet tramite impostazione della `referrerPolicy`, migliorando la compatibilità dei report HTML esportati, in particolare con Firefox.

### 🎨 Miglioramenti UI

- Introdotta una nuova interfaccia a schede per passare rapidamente tra Altimetria e Mappa 2D.
- Realizzato un layout dedicato alla Mappa 2D con sidebar, controlli, selezione dello stile cartografico, grafico altimetrico e pannelli informativi integrati.

### Risultato finale

Il report HTML esportato diventa un documento interattivo che integra sia l'analisi altimetrica sia la visualizzazione geografica della sessione. Quando sono disponibili dati GPS, è possibile navigare tra le due viste mantenendo tutte le funzionalità all'interno di un unico report esportato.
